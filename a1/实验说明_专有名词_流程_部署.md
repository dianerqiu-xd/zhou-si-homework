# 图像颜色空间与图像插值实验说明

本文档用于说明 `a1` 实验中的专有名词、整体实验流程，以及如何将代码部署到 Streamlit Community Cloud 获得公网 URL。

## 一、实验主题

本实验主题是：

```text
Vibe Coding 图像颜色空间、图像插值
```

实验使用 Python、OpenCV、NumPy、Matplotlib、Streamlit 等工具，实现以下功能：

1. 读取输入图像。
2. 输出 RGB、HSV、YCrCb、Lab 等颜色空间的各通道图像。
3. 手写 RGB 到 HSV 的转换算法。
4. 实现最近邻插值和双线性插值。
5. 支持图像放大、缩小、旋转、拉伸等操作。
6. 使用 Streamlit 做成可交互网页，并支持上传到 GitHub 后部署为公网 URL。

## 二、专有名词解释

### 1. 图像

数字图像可以理解为一个二维像素矩阵。每一个像素都有颜色值。例如一张 RGB 彩色图像中，每个像素通常包含三个数值：

```text
R 红色通道
G 绿色通道
B 蓝色通道
```

每个通道的值通常在 `0-255` 之间。

### 2. 像素

像素是图像的最小显示单位。图片越大，像素越多，能表达的细节也越多。

例如：

```text
640 x 480
```

表示图像宽度为 640 个像素，高度为 480 个像素。

### 3. RGB

RGB 是最常见的颜色空间，由三个通道组成：

```text
R = Red，红色
G = Green，绿色
B = Blue，蓝色
```

RGB 适合显示设备使用，例如电脑屏幕、手机屏幕。

本实验中会输出：

```text
RGB_R_gray.png
RGB_G_gray.png
RGB_B_gray.png
```

分别表示红、绿、蓝三个通道的灰度图。

### 4. BGR

BGR 是 OpenCV 默认读取图像时使用的通道顺序：

```text
B = Blue
G = Green
R = Red
```

而大多数显示工具使用 RGB 顺序，所以代码中读取图片后会执行：

```python
cv2.cvtColor(bgr, cv2.COLOR_BGR2RGB)
```

这样可以避免显示时颜色颠倒。

### 5. 颜色空间

颜色空间是一种描述颜色的方法。

同一张图片可以用不同方式表示，例如：

```text
RGB：按红绿蓝表示
HSV：按色相、饱和度、明度表示
YCrCb：按亮度和色度表示
Lab：按亮度和感知颜色差异表示
```

不同颜色空间适合不同任务。例如 HSV 更适合按颜色筛选目标，YCrCb 更适合分离亮度和颜色信息。

### 6. 通道

通道是图像某一个颜色分量的二维矩阵。

例如 RGB 图像有三个通道：

```text
R 通道
G 通道
B 通道
```

HSV 图像也有三个通道：

```text
H 通道
S 通道
V 通道
```

本实验会把每个通道单独保存成图片，方便观察各通道的作用。

### 7. 灰度图

灰度图只有一个通道，像素值表示亮暗程度。

```text
0   = 黑色
255 = 白色
```

本实验输出颜色空间通道时，会把单个通道保存成灰度图。

### 8. HSV

HSV 是一种更接近人类感知的颜色空间，由三个通道组成：

```text
H = Hue，色相
S = Saturation，饱和度
V = Value，明度
```

含义如下：

```text
H：表示颜色种类，例如红色、绿色、蓝色
S：表示颜色纯度，越高颜色越鲜艳
V：表示亮度，越高图像越亮
```

### 9. 手写 HSV 转换

“手写 HSV 转换”是指不直接调用 OpenCV 的：

```python
cv2.cvtColor(rgb, cv2.COLOR_RGB2HSV)
```

而是自己根据数学公式实现 RGB 到 HSV 的转换。

本实验中的核心函数是：

```python
rgb_to_hsv_manual(rgb)
```

基本步骤是：

```text
1. 把 R、G、B 从 0-255 归一化到 0-1
2. 对每个像素求 max(R, G, B)
3. 对每个像素求 min(R, G, B)
4. delta = max - min
5. 根据最大通道是谁计算 H 色相
6. S = delta / max
7. V = max
8. 将 H、S、V 转成图像可保存的 0-255 范围
```

这样做的意义是证明自己理解颜色空间转换原理，而不是只调用现成函数。

### 10. YCrCb

YCrCb 是一种把亮度和色度分离的颜色空间：

```text
Y  = 亮度
Cr = 红色色度差
Cb = 蓝色色度差
```

它常用于图像压缩、视频编码、人脸检测等场景。

### 11. Lab

Lab 是一种更接近人类视觉感知的颜色空间：

```text
L = Lightness，亮度
a = 绿色到红色方向的颜色分量
b = 蓝色到黄色方向的颜色分量
```

Lab 常用于颜色校正、图像增强、颜色差异分析。

### 12. 归一化

归一化是把数据缩放到统一范围。

例如 RGB 原始值是：

```text
0-255
```

计算 HSV 时通常先变成：

```text
0-1
```

这样更方便使用数学公式。

### 13. 图像插值

图像插值是指在图像尺寸或形状变化时，估计新像素颜色值的方法。

当图像被放大、缩小、旋转或拉伸时，新图像中的像素位置不一定能刚好对应到原图整数像素位置，因此需要插值算法来计算新像素值。

### 14. 最近邻插值

最近邻插值是最简单的插值算法。

做法是：

```text
目标图像中的一个像素反推到原图坐标后，
直接选择距离它最近的原图像素作为结果。
```

优点：

```text
计算速度快
实现简单
```

缺点：

```text
放大后容易产生块状感和锯齿感
```

本实验中的核心函数是：

```python
nearest_resize(image, new_width, new_height)
```

### 15. 双线性插值

双线性插值比最近邻插值更平滑。

做法是：

```text
目标图像中的一个像素反推到原图坐标后，
找到它周围 4 个像素，
根据距离进行加权平均。
```

优点：

```text
结果更平滑
视觉效果更自然
```

缺点：

```text
计算量比最近邻插值更大
```

本实验中的核心函数是：

```python
bilinear_resize(image, new_width, new_height)
```

### 16. 反向映射

反向映射是图像几何变换中的常见方法。

不是从原图像素推到目标图像，而是：

```text
遍历目标图像的每个像素
反推它在原图中对应的位置
再从原图采样得到像素值
```

这样可以减少目标图像中出现空洞的问题。

本实验中的缩放、旋转、拉伸都使用了反向映射思想。

### 17. 旋转

旋转是改变图像方向的操作。

本实验中，用户可以在 Streamlit 页面上调整旋转角度，例如：

```text
25 度
-30 度
```

旋转之后，原图像素不会刚好落在整数坐标上，因此需要最近邻或双线性插值。

### 18. 拉伸

拉伸是改变图像横向或纵向比例的操作。

例如：

```text
横向拉伸 1.2 倍
纵向压缩 0.8 倍
```

本实验中旋转和拉伸可以同时进行。

### 19. OpenCV

OpenCV 是常用的计算机视觉库。

本实验中 OpenCV 主要用于：

```text
读取图片
保存图片
辅助转换 YCrCb、Lab 颜色空间
```

代码中导入方式是：

```python

```

### 20. NumPy

NumPy 是 Python 中进行矩阵和数组计算的核心库。

图像本质上就是一个数组，所以本实验中大量使用 NumPy 进行像素级计算。

代码中导入方式是：


### 21. Matplotlib

Matplotlib 是 Python 中常用的绘图库。

本实验中它主要用于生成测试截图拼图，例如：

### 22. Pillow

## 四、核心代码逻辑

### 1. 核心算法脚本

核心源码位于：

```text
src/image_color_interpolation.py
```

整体逻辑是：

```text
读取图片
-> 转为 RGB
-> 计算颜色空间通道
-> 保存 RGB、HSV、YCrCb、Lab 通道图
-> 执行最近邻插值
-> 执行双线性插值
-> 执行旋转拉伸
-> 生成测试截图拼图
```

核心函数包括：

```text
read_rgb_image()
读取图像并转换为 RGB。

rgb_to_hsv_manual()
手写 RGB 到 HSV 的转换。

save_color_space_channels()
保存不同颜色空间的各通道结果。

nearest_resize()
实现最近邻缩放插值。

bilinear_resize()
实现双线性缩放插值。

rotate_and_stretch()
实现旋转和拉伸变换。

make_screenshots()
生成测试截图拼图。
```

## 十一、实验总结

本实验通过 Vibe Coding 的方式完成了一个完整的图像处理应用。算法部分实现了颜色空间拆分、手写 HSV 转换、最近邻插值、双线性插值、旋转拉伸等功能；工程部分通过 Streamlit 将算法包装成网页应用，并可以通过 GitHub 部署到 Streamlit Community Cloud。

通过本实验可以理解：

```text
1. 图像本质上是像素矩阵
2. 颜色空间是对颜色的不同表达方式
3. HSV 可以通过 RGB 数学公式手写实现
4. 图像缩放和旋转需要插值算法
5. 最近邻快但粗糙，双线性更平滑
6. Streamlit 可以快速把 Python 算法部署成在线网页
```
